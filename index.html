Bike Trip Planner

Go to Bike Trip Planner

Using the Google Maps API, my team created a bike trip planner. Users would enter their desired start and end positions into google maps. After getting the position and elevation data from Google, this data was used to create a 3D map of the elevation of the ride, thus showing the steepest parts of the ride. This enabled bike-riders to plan trips that would avoid steep hills, as well as the total amount of uphill and downhill distances that would be covered during the planned trip.

A web app that improves Google map routing for cyclists by enabling a cyclist to be routed on predefined safer streets, provide a route, total ascent/descent information with a colour coded gradient overlay for the route. Go to Bike Trip Planner

  • Technologies: Ruby, Ruby on Rails, PostgreSQL, CSS, HTML, CSS, JavaScript, Canvas, Bootstrap, Font Awesome
  • Date: December 2014

Angular Trader

Go to Angular Trader

Using plain javascript rather than a 3D library, I created a small but powerful 3D engine that was responsive and worked well on mobile. The eventual goal of this project is to add educational aspects to the game, make learning fun and engaging..

I created a custom 3D game engine for the app, without using any external libraries. All the animation code was written in vanilla JavaScript on the front-end. The code measures the performance of the client device, and adjusts the displayed detail to enable reasonable performance on lower end devices such as mobile phones. All graphics were drawn to the HTML5 canvas element, rather than using webGL. This ensured wider compatibility and simplified development, at the cost of performance. During development and testing it was found that performance could be increased by reducing calls to the draw method of the canvas context. So instead of drawing simple triangles as the graphics primitive as is standard, the engine draws fewer but more complex polygons. The number of polygons are reduced on lower end devices. Go to Angular Trader

  • Technologies: Ruby, Ruby on Rails, PostgreSQL, CSS, HTML, CSS, JavaScript, Canvas, Bootstrap, Font Awesome
  • Date: December 2014

VCE Further Mathematics Practise Exam

Go to VCE Further Maths

This app was designed to give on-line exam practise for VCE students.

The app was designed to look almost identical to the actual pdf of previous exams. However javascript and d3.js were used to add dynamic content to the page and aid learning.

Unlike paper practise exams, the app can randomise questions to enable further practise, as well as automatically mark questions correct and display how long it took besides other user stats. The questions are designed to use multiple choice, so students can practise them while on public transport.
